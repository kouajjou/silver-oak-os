🐛 Action 1 — Bug Alex Orchestrator FIXÉ

Diagnostic PhD

Mémoire disait : "Bug `alex_orchestrator` hardcode `dispatchToEmployee` non-fixé"

Réalité : la fonction dispatchToEmployee a déjà été refactorée pour utiliser delegateToAgent(). MAIS j'ai trouvé un autre bug subtil en testant en live :

Vrai bug détecté en live : classifyDomainRoute() utilisait first-match-wins. Donc "Rédige un post LinkedIn" matchait comms en premier (à cause de "rédige") AVANT que content soit testé (avec "linkedin"). Score 4/5 patterns au lieu de 5/5.

Fix appliqué (commit e89e62b0)

Algorithme score-based : on évalue TOUS les patterns et on garde celui avec le plus de matches.

function classifyDomainRoute(message) {
  let bestRoute = null;
  for (const entry of DOMAIN_ROUTES) {
    let score = 0;
    for (const pattern of entry.patterns) {
      const globalPattern = new RegExp(pattern.source, pattern.flags + 'g');
      const matches = message.match(globalPattern);
      if (matches) score += matches.length;
    }
    if (score > 0 && (!bestRoute || score > bestRoute.score)) {
      bestRoute = { intent: entry.intent, agentId: entry.agentId, score };
    }
  }
  return bestRoute ? { intent, agentId } : null;
}
