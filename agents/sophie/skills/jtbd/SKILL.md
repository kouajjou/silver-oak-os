# Skill: jtbd

**Agent**: Sophie (Product & UX)
**Priority**: P0

## Purpose

Apply the Jobs To Be Done framework to understand what customers are actually hiring the product to do.

## When to use

Early discovery phase, before writing any PRD.
Especially useful when Karim thinks he knows the feature but hasn't validated the underlying job.

## Framework

### The JTBD statement
"When [situation], I want to [motivation/job], so I can [outcome/progress]."

### 3 job layers
1. **Functional** — the practical task (e.g., "track my expenses")
2. **Emotional** — how they want to feel (e.g., "feel in control of my money")
3. **Social** — how they want to be perceived (e.g., "look like a responsible adult")

### Forces diagram
- **Push** (away from current solution): what's wrong with how they do it now?
- **Pull** (toward new solution): what's attractive about the new approach?
- **Anxiety** (holding them back): what's scary about switching?
- **Habit** (gravitational pull): what keeps them doing it the old way?

## Output format

```
🎯 JTBD ANALYSIS: [product name]

FUNCTIONAL JOB: [one sentence]
EMOTIONAL JOB: [one sentence]
SOCIAL JOB: [one sentence]

FORCES:
→ Push: [2-3 bullets]
→ Pull: [2-3 bullets]
→ Anxiety: [2-3 bullets]
→ Habit: [2-3 bullets]

KEY INSIGHT: [one actionable insight for product design]
```
