### 1. Complétude
- [ ] Architecture décrite et à jour

- [ ] Testing strategy

### 2. Exactitude
- [ ] Les chemins de fichiers existent encore
- [ ] Les commandes fonctionnent
- [ ] Les ports/URLs sont corrects

- [ ] Les patterns de code correspondent au code réel

### 3. Efficacité
- [ ] Pas de sections redondantes
- [ ] Pas d'informations obsolètes
- [ ] Instructions claires et actionables
- [ ] Exemples de code utiles (pas trop, pas trop peu)

- [ ] Les instructions sont non-ambiguës
- [ ] Les priorités sont claires
