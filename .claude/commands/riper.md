Execute the RIPER methodology on the following task. RIPER = 5 phases strictes:

## Phase 1 — RESEARCH (Recherche)
- Lire et comprendre TOUT le code concerné avant de toucher quoi que ce soit
- Identifier les fichiers, fonctions, dépendances impliqués
- Documenter l'état actuel (ce qui existe, ce qui marche, ce qui ne marche pas)
- NE RIEN MODIFIER dans cette phase

## Phase 2 — INNOVATE (Innover)
- Proposer 2-3 solutions possibles
- Pour chaque solution: effort estimé, risques, impact sur l'existant
- Recommander la meilleure solution avec justification

## Phase 3 — PLAN (Planifier)
- Décomposer la solution choisie en étapes atomiques
- Lister chaque fichier à créer/modifier avec les changements précis
- Identifier l'ordre d'exécution (dépendances entre étapes)
- Définir les critères de succès (tests, validation)

## Phase 4 — EXECUTE (Exécuter)
- Implémenter chaque étape du plan, une par une
- Après chaque étape: vérifier que rien n'est cassé
- Commit intermédiaires si approprié

## Phase 5 — REVIEW (Revue)
- Relire tout le code modifié
- Vérifier: pas de régressions, pas de code mort, pas de secrets exposés
- Lancer les tests
- Résumé final des changements
