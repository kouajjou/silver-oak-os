Explique le code spécifié en détail, adapté au niveau du fondateur solo.

## Cible
$ARGUMENTS
(Fichier, fonction, module, ou concept à expliquer)

## Format d'explication

### 1. Vue d'ensemble (30 secondes)
- Qu'est-ce que ce code fait, en une phrase simple
- Pourquoi il existe (quel problème il résout)
- Où il s'inscrit dans l'architecture Claudette by Silver Oak

### 2. Comment ça marche (2 minutes)
- Flux de données étape par étape
- Les entrées et sorties
- Les dépendances externes (APIs, DB, autres services)
- Diagramme ASCII si utile

### 3. Points importants
- Les parties critiques (sécurité, performance, fiabilité)
- Les patterns utilisés et pourquoi
- Les pièges potentiels

### 4. Connexions
- Quels autres fichiers/services utilisent ce code
- Ce qui se casse si on le modifie
- Les tests qui le couvrent

### 5. Améliorations possibles
- Quick wins identifiés
- Dette technique éventuelle
- Suggestions (uniquement si pertinent)
